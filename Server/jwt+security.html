jwt+spring security

jwt+spring security

jwt

认证流程：

1.用户使用用户名和密码请求登录；
2.服务端收到请求，验证用户名和密码；
3.服务端生成JWT，并返回给客户端；
4.客户端保存JWT，并在之后的请求中携带JWT；
5.服务端收到请求，验证JWT的合法性和有效性；
6.服务端处理请求，并返回响应给客户端。

核心是在第3步和第5步：

spring security

主要功能：认证 + 授权。

流程：

1.注册filter

2.自定义provider进行认证

3.自定义认证逻辑

自定义范围非常之广：

jwt token代码使用流程：

1.集成jwt token代码。
2.配置header key，加密key，过期时间
3.定义JwtTokenUtils，包括创建token，验证token，解析token的方法。
4.定义filter，默认解析token，并保存user信息到SecurityContext(本质是一个ThreadLocal)。 这里是结合了spring security，当然可以不这样结合。

微信小程序登录流程：

1.进入登录页，用户调用 wx.login() 获取code
2.请求服务端login接口，传入code。
3.服务端请求微信后台，包含code, appkey，appsercet返回openid， sessionKey
4.查询用户信息，如果无，则创建用户信息。
5.根据openid创建jwt token
6.后续流程大概就是请求，验证token，而不用手动传uid了。
