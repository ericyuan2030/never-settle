Tomcat启动流程

Tomcat启动流程

tomcat解析流程：https://blog.csdn.net/qq_38975553/article/details/103443321

启动入口： NioEndpoint.startInternal() 方法。

初始化顺序是反着来的。 
Acceptor线程: 开始接收客户端请求, 接收到请求后交由pollerThread进行轮询判断该socket是否可读


PollerThread线程: Acceptor接收到请求后会交给Poller处理, poller线程将会进行轮询select, 一旦socket可读, 将该socket扔给Executor处理

1.客户端发送请求先到tomcat

2.tomcat初始化接收线程，poller线程，线程池进行处理请求。 EndPoint。

3.通过coyoteAdapter将socket请求信息封装为request，传递给container。

4.container是一个容器，包含 Engine, host, context, wraper。有非常多的容器，每个容器处理之后传给下一个容器，同时构成一个责任链，一个处理完之后交给下一个。

5.最后交给standardWrapper，从配置文件创建加载serverlet，一般是dispathcServelet. (从web应用程序的WEB-INF/classes或者WEB-INF/lib目录中加载类文件)

6.standardWrapper调用invoke方法，

if (!unavailable) {
    servlet = wrapper.allocate();
}

if (request.isAsyncDispatching()) {
    request.getAsyncContextInternal().doInternalDispatch();
} else {
    filterChain.doFilter(request.getRequest(),
            response.getResponse());
}

前者调用dispatch方法，最后还是会转给 ApplicationFilterChain。

后者在 ApplicationFilterChain #internalDoFilter调用了：

servlet.service(request, response); 最后转给 dispatch。

入口在tomcat.start()

最后的出口在 HttpServelet.service()方法。

所以tomcat的本质就是接收socket信息，然后封装成request交给spring框架？nginx是不是也是类似的。
