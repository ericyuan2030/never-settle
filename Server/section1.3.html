集合：有点类似table的概念，将多个有关联或者类似结构的文档放到一个地方。

//table级别

优化部分：

1.创建索引
  db.col.createIndex({"title":1})
  db.col.createIndex({"title":1,"description":-1})
2.管道聚合
  db.articles.aggregate([
                      { $match : { score : { $gt : 70, $lte : 90 } } },
                      { $group: { _id: null, count: { $sum: 1 } } }
                       ]);
  $match用于获取分数大于70小于或等于90记录，然后将符合条件的记录送到下一阶段$group管道操作符进行处理。
3.主从复制同步。
  读数据一定是从主节点读取，不会有主从不同步的问题。
4.支持分布式操作。
5.高级查询（嵌套查询）：

mongodb四期

使用 + 原理

MongoDB本身是一种数据库，支持增删改查操作。

基础概念

  • 文档：就是JSON格式的键值对
  • 集合：类似于关系型数据库中的表，将多个有关联或类似结构的文档放到一个地方

核心API

  • 表级别：
    • show collections;
    • db.xxxcollection.drop()
  • 基础增删改查：
    • db.collections.find()
    • db.collections.insert()
    • db.collections.update(query, update)
    • db.collections.save()
    • db.collections.remove(query), deleteOne(), deleteMany()
  • 条件查询：
    • db.col.find({"likes": {$gt:50}, $or: [{"by": "菜鸟教程"},{"title": "MongoDB 教程"}]}).pretty()
  • 类型查询：
    • db.col.find({"title": {$type : 2}}) 或 db.col.find({"title": {$type : 'string'}})
  • 排序和限制：
    • db.COLLECTION_NAME.find().sort({KEY:1})

优化部分

  1. 创建索引：
    • db.col.createIndex({"title":1})
    • db.col.createIndex({"title":1,"description":-1})

  2. 管道聚合：

     db.articles.aggregate([
     { $match : { score : { $gt : 70, $lte : 90 } } },
     { $group: { _id: null, count: { $sum: 1 } } }
 ]);

  3. 主从复制同步。

    读数据一定是从主节点读取，不会有主从不同步的问题。

  4. 支持分布式操作。

  5. 高级查询（嵌套查询）：

     var result = db.users.findOne({"name":"Tom Benzamin"},{"address_ids":1})

  6. 事务操作：

     db.books.findAndModify ( {
     query: {
         _id: 123456789,
         available: { $gt: 0 }
     },
     update: {
         $inc: { available: -1 },
         $push: { checkout: { by: "abc", date: new Date() } }
     }
 })

  7. Map函数

  8. 支持正则表达式

Java使用

  1. Document
  2. MongoCollection（引用类型变量，直接更新、删除）

总结

MongoDB本质上是一种数据库，支持增删改查操作。相比传统的关系型数据库，MongoDB更为灵活，同时也支持增删改查，而且上层使用了函数式的方式，更靠近代码，而不是SQL语言。核心是document和collection。

本页总结

  1. MongoDB是一种数据库，支持增删改查操作。
  2. 核心是document和collection。
  3. 主要API包括基础增删改查和条件查询，同时支持高级查询、管道聚合、创建索引等操作。
