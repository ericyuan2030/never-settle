分布式+集群

分布式

分布式和微服务的区别，分布式更注重负载均衡，容错处理、服务分发相关的知识。

微服务其实是把一个大的系统拆分成多个服务。

redis 分布式锁

1. 最简单的代码： 枷锁成功，设置过期实际，finally进行释放

if (setnx(key, 1) == 1){
    expire(key, 30)
    try {
        //TODO 业务逻辑
    } finally {
        del(key)
    }
}

2. 遇到的问题：

1. 非原子性
   服务器挂掉，没有设置过期实际，导致锁一直未被释放。
   使用lua脚本解决

2. 误解除：A枷锁，未执行完，B获取锁，A执行完，释放锁。
   使用lua脚本验证是否一致，加锁和解锁的是否是一个线程。

3. 超时引起并发
   30s内未操作完，自动解除了。
   增加守护进程，自动续锁。

4. 可重入锁


5. 无法获取锁，等待时间长，发布订阅模式。

1. 主备切换


2. 集群脑裂

总结

Redis 以其高性能著称，但使用其实现分布式锁来解决并发仍存在一些困难。Redis 分布式锁只能作为一种缓解并发的手段，如果要完全解决并发问题，仍需要数据库的防并发手段。
