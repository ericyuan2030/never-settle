线程

线程

锁池和等待池的区别：

1. synchronized方法进行阻塞，线程会进入锁池；当释放锁之后，锁池的线程才能进行竞争。
2. A调用了wait()方法之后，A会释放锁，进入对象的等待池，只有其他线程调用了notify，才会从等待池移动到锁池里面。

什么情况下会进入阻塞状态：

1. 调用wait()方法，进入等待池。
2. synchronized 线程会进入锁池。
3. sleep join 会进入阻塞状态。

sleep wait join yield区别：

1. Thread.sleep() 释放cpu，不释放锁，进入阻塞状态
2. Object.wait，本身是在synchronized里面，释放cpu，释放锁，进入锁池，只有notify之后才会进入等待池。
3. t.join() ,当前线程阻塞，等待t线程完成之后。Waits for this thread to die
4. yield() 进入就绪状态，等待cpu重新分配调度。

构建线程的方式：

1. 继承Thread，重写run方法，然后直接 new MyThread().start();
2. 创建匿名内部类， new Thread(new Runnable(){}).start()
3. 构建callable，然后封装成FutureTask，（这个即时runnable，又是callable），然后传给Thread，然后可以调用 futureTask.get()获取返回值
4. 所以，是的，只要你实现了一个接口或者一个抽象类的方法，并在这个过程中创建了一个对象，你就创建了一个匿名内部类。这种方法在处理简单的接口实现或者需要覆盖很少方法的抽象类时非常有用，因为它可以减少编写额外类的需要。

线程池参数：

1. 核心线程
2. 非核心线程
3. 工作队列
   超过工作队列，小于最大线程的时候，创建新的线程，默认是执行新提交的任务。
4. 最大线程
5. 存活时间
   1.非核心线程能够被回收
   2.设置 allowCoreThreadTimeout = True 的核心线程
6. 拒绝策略
   1.abort，丢弃抛出异常。
   2.discard, 丢弃不抛出异常
   3.discardOldest, 丢弃最早的
   4.CallerRun()，由调用线程执行

线程池阻塞队列配置 

1. ArrayBlockingQueue
2. LinkedListBlockingQueue
3. PriorityBlockingQueue
4. SynchronousBlockingQueue
   1. 队列不存储元素，队列长度为0.如果调用put，则线程A被阻塞，必须等另外一个线程B执行take，才能取消A的阻塞。
   2. 支持栈和队列，默认是队列，构造函数传入 nonfair，默认false
   3. newCachedThreadPool就是使用这个，代表任务希望被快速处理。
